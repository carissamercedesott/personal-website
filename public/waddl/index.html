Waddl

A cozy desktop window where Claude Code waddles through your codebase, narrating as it reads and edits. No terminal required — that's the point.

AI Agents Electron TypeScript In progress
01

The problem

An AI coding agent does most of its work where you can't see it: it reads ten files, plans, edits three of them, and what you get is a terminal scrolling by. That's fine if you live in terminals. Most people don't.

Waddl is my running answer to one question: how do you make an AI coding agent's activity legible without a terminal? The bet is that legibility is an interface problem, not a logging problem — the agent's activity needs to be read, the way a progress bar or a diff is read, not scrolled.

03

Three problems, three fixes

The thesis breaks into three concrete problems — the specific places an agent stops being legible. None of these are new problems, and other tools take a swing at each. What follows is the interaction Waddl uses, drawn as it renders in the app.

Every chat lands in one flat list

Agent sessions pile up faster than any other kind of chat — one throwaway question and one week-long refactor look identical in the list. Three projects' worth end up in a single column sorted by recency, so finding the session where you fixed the theme tokens means scrolling and guessing at titles the agent wrote for you.

Waddl's fix Project folders. Chats live in named, collapsible folders you drag them into. Collapse the projects you're not in; anything unfiled stays visible under ungrouped rather than vanishing into a default bucket you forget to check.

Undoing an agent is all-or-nothing

An agent makes six edits and the third one was wrong. Usually your options are keep everything or throw it all out. Git will get you back, but only if you thought to commit before you started — and the point of this window is that you shouldn't need to think in git to use it.

Waddl's fix A rewind rail. Every file edit drops a checkpoint in the timeline's own gutter — the rewind lives where the work happened, not behind a command you have to know about. Hover a checkpoint and the steps it would undo dim in place, so you see the cost before you pay it. Click, and the tail folds away rather than being deleted.

Edits arrive as a wall of diff

Ten files of red and green is technically shown, but it isn't legible — the one line that matters is buried in the noise, and the context lines you already agree with take up most of the screen.

Waddl's fix Collapsible diffs, written as they land. Each file is a card that collapses to a one-line summary, and its rows stream in as the edit actually happens — so an edit is something you watch arrive, not a wall that appears at the end. Untouched runs fold to a single line you can open in place when you want to check them.
