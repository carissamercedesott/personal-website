07 · Colophon

How this site is built

The site is its own case study — every choice below is one I'd defend in a design review.

Static first, framework never

Every page is hand-written HTML and CSS. Nothing hydrates, and nothing ships that isn't used — the fastest JavaScript is the JavaScript that never loads.

One token sheet under everything

Color, type, spacing, radii, elevation, and motion all live in tokens.css. The design-system page renders its palette by reading those variables live in the browser, so the docs physically can't drift from the source.

Interactivity as Solid islands

The lab demos are TypeScript + Solid components, compiled with Vite and mounted only where they appear. The 3D smiski goes further — three.js and its physics engine load lazily, and only if you scroll to him.

Motion with manners

Every animation runs on the easing and duration tokens and steps aside for prefers-reduced-motion — and every interaction works with a keyboard alone.

A turntable, synthesized

The corner record player's lo-fi loop is pure Web Audio — oscillators, filtered noise, vinyl crackle from a sparse noise buffer. No samples. Drag the BPM and the whole site keeps time.
